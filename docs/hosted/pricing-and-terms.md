# Hosted Pricing And Terms

This is the public-facing page for hosted `email-to-telegram` pricing and
commercial terms. It is an operator/product draft, not legal advice.

> **Status:** Hosted is in donation-only beta. There is no live billing, no
> paid plans, and no self-serve upgrade flow. Any structured paid plan
> ("Personal" / "Pro" / "Team" / "Business") referenced in older drafts is
> **not in effect.** The only ways to support hosted today are donations and
> personal arrangements (see below).

## Self-Hosting

The project is MIT-licensed. Self-hosting is free, well-documented, and the
recommended path for serious or high-volume use. See the repository README
for setup.

## Hosted Today

Hosted is a single free tier for everyone, supported by donations.

- One inbound shared domain (operator-managed).
- Soft daily abuse-prevention rate limit: **10 alias creation attempts per
  Telegram user per day.**
- No automated billing, no per-plan caps. Reasonable use is welcome; bulk,
  marketing, or shared-domain-reputation-risky use is not — see
  [`acceptable-use.md`](./acceptable-use.md).
- Self-serve account wipe (`/delete_me`) and data export (`/export_me`) are
  live — see [`privacy-and-data-requests.md`](./privacy-and-data-requests.md).

## Future Paid Plans

Structured paid plans with self-serve checkout, customer portal, automated
billing, tax handling, and refund flows are a future direction — not a
present offering. They will be announced when the operator entity, payment
provider, and tax setup are in place. Until then, all paid-style
arrangements happen through the Custom Arrangements path above.
