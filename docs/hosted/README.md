# Hosted Policy Drafts

These drafts are for the managed hosted service. They are product/operator
drafts, not legal advice. Replace placeholders, review with the right legal/tax
advice for your jurisdiction, and revisit them when paid billing is enabled.

> **Status:** The hosted instance currently runs in beta with no live billing.
> All paid-plan mechanics (checkout, customer portal, webhooks, refund flows,
> tax handling) are **to be implemented.** These drafts describe the planned
> shape so contributors and prospective hosted users can review direction
> early.

## Drafts

- [`pricing-and-terms.md`](./pricing-and-terms.md): planned pricing, plan
  limits, cancellation, and refund direction
- [`acceptable-use.md`](./acceptable-use.md): public acceptable-use and abuse
  reporting language
- [`privacy-and-data-requests.md`](./privacy-and-data-requests.md): hosted
  privacy and data request language

## Required Public Contacts

Before live public signup, the following placeholders must be replaced
everywhere:

- `<abuse@example.com>` for abuse reports and shared-domain reputation issues
- `<support@example.com>` for general support
- `<privacy@example.com>` for privacy, export, and erasure requests

These contact addresses are **to be configured** as part of the live hosted
launch. The drafts are published in their pre-launch form to make the planned
policy visible early.
